PRINCIPAIS COMANDOS DO GIT

Comandos Básicos

  • cd nomeDoCaminho : usado para entrar nas pastas do computador
  • cd .. : serve para retornar uma pasta
  • ls: serve para visualizar o que tem na pasta em que você se encontra
  • git config --list : exibe a lista de configurações atuais do git
  • touch nomeDoArquivo.txt : cria um arquivo txt, docx ou rtf
  • nano nomeDoArquivo : abre um editor para editar o arquivo. *OBS: Para sair do editor utilize CTRL + X
  • git init : incia o Git na pasta que você está
  • git status : verifica o status dos arquivos presentes na atual posição que não foram commitados. *OBS: você precisa estar em uma pasta que tenha um repositório git, ou seja que tenha sido iniciado o "git init"
  • git add nomeDoArquivo : coloca o aquivo na área intermediária(staging), que fica antes de commitar o arquivo
  • git add . : envia todos os arquivos para a área intermediária(staging) ao mesmo tempo
  • git commit -m 'mensagem sobre o que foi alterado' : aqui você finalmente estará commitando o arquivo e escrevendo junto uma mensagem para o resto do time ler

Visualizando os logs

  • git log : podemos ver os commits que aconteceram
  • git log --stat : podemos ver os commits com mais detalhes(alterações, inserções e deletamentos)
  • git log --pretty=oneline : podemos ver os commits resumidamente em 1 linha

Visualizando as diferenças

  • git diff : mostra o que foi modificado nos arquivos
  • git diff nomeDoArquivo : mostra o que foi modificado no arquivo específico
  • git diff --name-only : mostra o nome dos arquivos que foram modificados

Desfazendo alterações

  • git commit --amend : permite editar o último commit feito adicionando os arquivos que estão em staging e permite alterar a mensagem escrita no commit. *OBS: se você quiser modificar apenas a mensagem do commit utilize o mesmo comando sem ter arquivos na área staging
  • git reset HEAD nomeDoArquivo : serve para retirar o arquivo de staging, ele vai continuar com as alterações, mas não vai poder receber um commit até que ele seja adicionado em staging de novo
  • git checkout -- nomeDoArquivo : volta o arquivo para como ele era no último commit

Lidando com branchs

  • git branch : mostra quais as branchs existem no projeto. *OBS: a branch "master" já vem ao criar o projeto
  • git branch nomeDaBranch : cria uma nova branch
  • git checkout nomeDaBranch : vai para a branch digitada
  • git branch -d nomeDaBranch : deleta a branch digitada
  • git checkout -b nomeDaBranch : cria e altera para uma nova branch
  • git merge nomeDaBranch : Pega os dados da merge digitada e mescla com a merge da atual posição

Clonando um projeto

  • git clone linkDoProjeto : clona qualquer projeto através do seu link retirado do github e conecta o projeto da nuvem com o do diretório em que foi executado o comando

Subindo um projeto para o GitHub

  • git remote add origin linkDoNovoProjetoNaNuvem : conecta o projeto da máquina do usuário com o projeto na nuvem, no caso, o GitHub. OBS*: o link para conectar os dois está na página do novo projeto no github
  • git push origin nomeDaBranch : esse comando sobe de fato o projeto para a origem(nesse caso, o github). *OBS: "origin" refere-se a origem setada pelo comando de cima(o link para onde vai o projeto). *OBS 2: "nomeDaBranch" refere-se para qual branch do projeto na nuvem vai os arquivos

Baixando as atualizações de um projeto

  • git pull origin nomeDaBranch : baixa os códigos da branch digitada da origin(link do projeto no github). *OBS: caso os arquivos estejam exatamente igual vai aparecer "Already up to date."
